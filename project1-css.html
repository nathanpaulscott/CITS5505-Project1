CSS - Purpose and Motivation

What is CSS

CSS (Cascading Style Sheets) is a text based language used to primarily control how content is rendered on HTML webpages. It doesn't care about the content of an HTML webpage, it only cares about how it can assign styling to the elements of the webpage. CSS is the second member of the holy trinity of the web, i.e. HTML-CSS-Javascript. The syntatic construct is:

selector {
    style_attribute: value;
    style_attribute: value;
    style_attribute: value;
    . . . . .
    style_attribute: value;
}

Content and Presentation

Originally HTML had rudimentary styling capabilities. It became clear that managing larger complex websites with in-line specification of style in each tag was problematic on various levels from code modularity to style consistency, not to mention that the limited styling available in HTML made all websites look stylistically similar and boring.

See this page without CSS.

By introducing CSS and allowing style to be configured from a separate language and assigned to tags via selectors, the separation of content from presentation of that content could occur. This was, and still is, one of the key motivations behind CSS. Content doesn't really need to know about presentation and vice-versa, they are fundamentally different and somewhat orthogonal apects of the whole. In real-world companies, the presentation of a company's web presence may be designed and managed by different people than the content.

In terms of front-end web design, this separation can hugely improve the manageability of complex website code and enable precise control of presentation across various websites. It also allows vastly more presentation options over that of HTML. Another motivation behind CSS is the ability to render the same content in a multitude of ways depending on user variables (eg. screen size, disabilities, browser). This is beneficial as it moves all the complexity of adjusting presentation out of HTML. This alone would be enough of a reason for CSS, as designing a website without CSS to work seamlessly over any end-user device out there would be very difficult.

Selectors

The key to CSS is the selector. CSS selector language is a way of filtering out a subset of tags from the HTML document for the application of some particular styling parameters. We can basically select any tag via a combination of 8 selection paths:

  • tag selectors - select all tags of a specific type, eg.
    //selects any p tag
    p {
        color: red;
    }
    Note that we can also apply wildcards like * to the selector to widen the tag selection further.
  • group selectors - select all tags in the given comma separated list of tags, eg.
    //selects any p, span or i tag
    p, span, i {
        color: red;
    }
    Note that we could put any complex selector in a comma separated list, not just simple tags.
  • class selectors - select all tags that have a specific class assigned, eg.
    //selects any tag with class="some_class"
    .some_class {
        color: red;
    }
  • id selectors - select all tags that have a specific id, eg.
    //selects any tag with id="some_id"
    #some_id {
        color: red;
    }
  • attribute selectors - select all tags that have a specific attribute value, there are many attributes that can be selected apon, eg.
    //selects any tag with the attribute name="some_name"
    [name="some_name"] {
        color: red;
    }
  • contextual selectors - select all tags based their postition relative to other tags in the HTML tree heirarchy, eg.
    //p has to be a descendant / child of a div
    div p / div > p {
        color: red;
    }
    //p has to be an adjacent / general sibling of a div
    div + p / div ~ p {
        color: red;
    }
  • pseudo-class selectors - select all tags that have a specific state or relative position or property, eg.
    //selects any h2 tag that is currently in hover state, i.e. the mouse is positioned over the element
    h2:hover {
        color: red;
    }
    /*selects any p tag that is the 3rd child of it's parent*/
    p:nth-child(3) {
        color: red;
    }
  • pseudo-element selectors - select parts of tags, eg.
    //selects the point just after the content of all p tags, so if you add content there, it gets this style
    p::after {
        color: red;
    }
    //selects the part of the p tag that is currently selected by the user and applies this style
    p::selection {
        color: red;
    }
Tie-Breaking

The CSS cascading application rules are another critical part of CSS functionality. When multiple CSS styles apply to an element, eg. the color style of a div tag. The following precedence rules apply in order of importance:

  1. user-origin CSS
  2. inline CSS
  3. document level CSS
  4. external CSS

In the case of same level ties, the type matters (eg. id beats class) and in the case of same level and type ties, the latest position wins.

The Box Model

The last critical feature of CSS is the box model, where all tags have a layered box surrounding the inner content. The box model consists of:

  • Content - this is the innermost box and where the actual content is located
  • Padding - the content padding, it is styled as the content area and is there to add space
  • Border - the boundary between the inner content/padding area and the outer blank margin layer. The border can be visible or hidden and comprehensively styled
  • Margin - the transparent spacing between the border and the outermost postion of the tag which butts up against other outer margins of other tags
